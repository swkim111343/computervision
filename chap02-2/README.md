## 컴퓨터비전 챕터02-2 과제
1. 비주얼 스튜디오 프로젝트 속성에서 Debug 모드, Release 모드의 차이를 설명하라.
    - 먼저 Debug 모드는 프로그램 개발 과정에서 오류를 찾고 수정하기 위해 사용하는 빌드 모드이다.
    - 특징으로는 Debug는 디버깅 정보를 포함하여 프로그램을 빌드하고, 변수 값 확인, 중단점 사용 등이 가능하다.
    - 개발자가 프로그램의 오류를 찾고 수정하기 쉽도록 돕기 위한 모드이다.

2. 비주얼 스튜디오 프로젝트 속성에서 x86 모드, x64 모드의 차이를 설명하라.
    - x86 모드 : 32비트(32-bit) 환경에서 실행되는 프로그램을 생성하는 빌드 모드이다.
    - x64 모드 : 64비트(64-bit) 환경에서 실행되는 프로그램을 생성하는 빌드 모드이다.
    - 두 모드는 cpu 구조와 메모리 사용 범위에 차이가 있고, 프로그램의 실행 환경에 맞게 선택하여 사용하면 된다.

3. 빌드 전에 비주얼 스튜디오에서 포함 디렉토리를 설정하는 이유를 설명하라.
    - 헤더 파일을 찾기 위해 포함 디렉토리르 설정된다.
    - 포함 디렉토리가 설정되지 않으면 컴파일러가 헤더 파일을 찾지 못해 오류를 일으킨다.

4. 빌드 전에 비주얼 스튜디오에서 라이브러리 디렉토리를 설정하는 이유를 설명하라.
    - 라이브러리 파일 위치를 찾기 위해 설정한다.
    - 라이브러리 디렉터리를 설정하지 않으면 링커가 필요한 라이브러리 파일을 찾지 못해 오류가 발생한다.

5. 빌드 전에 비주얼 스튜디오에서 링커 추가종속성을 설정하는 이유를 설명하라.
    - 필요한 라이브러리를 링커에 지정하기 위해 추가 종속성을 설정한다
    - 그렇지 않으면 프로그램에서 사용한 함수의 실제 정의를 찾을수가 없어 오류가 발생한다.

6. DLL를 사용하기 위하여 시스템 환경변수 Path에 추가하는 경로는 무엇이고 왜 추가해야 하는지 설명하라.
    - 동적 링크 라이브러리 : C:\opencv\build\x64\vc16\bin 추가 해야함 
    - 위의 폴더에는 프로그램 실행시 필요한 .dll파일이 있음
    - path에 추가 해야하는 이유는 실행시 DLL 파일을 찾기위해 추가해야 한다.
